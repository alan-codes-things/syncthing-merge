# Syncthing Merge (`syncthing-merge`)

A cross-platform command-line tool designed to automatically scan, resolve, and merge Syncthing `.sync-conflict` files.

It utilizes Git's robust three-way merge engine to combine edits automatically when possible. When overlapping conflicts exist—or if no common ancestor is found—it launches **Meld** to let you perform a visual merge.

---

## How It Works

1. **Scan**: The tool recursively searches your active directory for files matching Syncthing's conflict pattern (`*.sync-conflict-*.*`), skipping system directories like `.git` and `.stversions`.
2. **Ancestor Lookup**: It looks in the `.stversions` directory for the most recent backup of the file to use as a **common ancestor** (the base).
3. **Merge Attempt**:
   - **If an ancestor is found**: It runs `git merge-file` to silently auto-merge the files. If the merge is clean, the conflict file is deleted automatically.
   - **If overlapping conflicts occur**: It opens Meld in **3-way mode** (Left: Base, Middle: Working file, Right: Conflict). Any edits and saves you make inside the middle pane apply directly to your working file.
   - **If no ancestor is found**: It opens Meld in **2-way mode** (Left: Working file, Right: Conflict). You can manually pull changes into your working file and save.
4. **Cleanup**: After you close Meld, the tool prompts you in the command line to confirm if the conflict was successfully resolved so it can safely delete the conflict backup file.

## Prerequisites

To run this tool, the following dependencies must be installed on your system:

1. **Git**: Must be globally accessible from your system PATH.
2. **Meld**: A visual diff and merge tool.
   - **macOS**: Installable via Homebrew (`brew install meld`).
   - **Linux**: Installable via your package manager (e.g., `sudo apt install meld`).
   - **Windows**: Download the installer from the official website and make sure to check the box to **"Add Meld to system PATH"** during setup.
3. **uv**: Fast Python packaging and tool manager.
